FragebogenPi – lokale Patientenaufnahme auf dem Raspberry Pi.

Anamnesebögen, Patientenfotos und Befundbilder mit GDT-Import und Rückschrieb in die Praxesoftware (z. B. T2med).
Vollständig offline. Vollständig isoliert. Open Source.

GDT Anamnese Patientenfoto Befundfotos Offline Isoliertes WLAN
GitHub Repository Dokumentation
▶ Installation in 3 Zeilen
Tipp: Script vorher ansehen (GitHub). 
wget https://fragebogenpi.de/install.sh
chmod +x install.sh
sudo bash install.sh

Benötigt

  • Raspberry Pi ab Version 3 (Plastikgehäuse empfohlen – wichtig für WLAN-Reichweite)
  • Raspberry Pi OS (Lite empfohlen)
  • High-Endurance-SD-Karte
  • iPad für die Anamnesebögen
  • optional: Smartphone für Befundfotos
  • optional: iPhone für Patienten-Selfies
  • LAN-Anschluss in der Nähe des Wartebereichs (für stabile WLAN-Abdeckung)
GDT-Import & Rückschrieb · Anamnesebögen · Befundfotos · Patienten-Selfie · vollständig offline · vollständig isoliert · Raspberry-Pi-Appliance · Open Source

Ablauf

  1. Patientendaten werden per GDT an FragebogenPi übertragen
  2. Patient füllt Formulare am iPad aus
  3. Rückschrieb der Daten und Bilder ins Praxisverwaltungssystem
